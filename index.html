| Headings

Headings are used to help organize the page to make it easier to read. They take up their own blocked area, and are bigger and bolder than normal text. You'll often see these be used as titles for articles, section headings, and other stuff like that. <h1> is the biggest, while <h6> is the smallest! They're pretty easy to use too - you just need to use an opening tag and a closing tag, and put the text in between the two tags!
NOTE: it is often good practice to make sure your headings are ordered well. For example, try to avoid doing something like using an <h1> tag, then an <h4> tag, then go back up to <h2>. Instead, go in the order of <h1>, <h2>, <h3>, then <h4>. At the very least, always start with an <h1> tag. There are a few reasons why we do this:

  • From a design standpoint, keeping headings in order makes it easier to read. It's like if you were to read a family tree that went from grandparents, to children, then parents. That ain't right. It's just confusing.
  • It helps people who use screen readers! This is generally people who have visual, mobility, and cognitive impairments. Properly ordered headings help these people understand the structure of the webpage. Some screen reader users heavily rely on using shortcuts to jump from heading to heading.
  • It helps improve your website's Search Engine Optimization (SEO) ranking. Search engines like Google rely on page headings to understand the page's topic and how the information is optimized. SEO is an entire thing of it's own, but in short, the better your website's SEO, the higher it will show on a results page for a search engine. Great if you want more people to view your website!

| Paragraphs

Paragraphs are simply your basic bodies of text, like what you're reading now. They are used by simply having an opening tag with a closing tag. ...hm. That's really all there is on paragraphs. Oh! I should mention that HTML documents don't really treat whitespace in your code like they would in a word processor like Microsoft Word. If you want to create a new paragraph, you would have to use the <br> tag to break up your text. It also is a self-closing tag, so you don't need to add a closing tag. Pretty cool, huh?

| Lists

There are two kinds of lists in HTML: Ordered lists and Unordered lists. Ordered lists create a list that is ordered. (Duh.) These lists have the items numbered as you go down the list. Unordered lists, on the other hand, use bullets for each item on the list. However, both lists are placed into HTML code the same way. You start with the opening and closing tags, <ul> and <ol>, for unordered lists and ordered lists specifically. These tags let the computer know that you want to make a list. Inside those tags, you use opening and closing tags for the element <li>, or "list item". You add another list item element for each item you want to add to the list!

| Images

You probably noticed that there were images on this website. That's thanks to the <img> tag! It's pretty self explanatory on what it does. It is written a bit differently though. First off, it's self-closing, so you don't need a closing tag. Second, you need to let the computer know what image you're trying to put in. This is done through the src, or source, attribute. Inside the opening tag, put src="Link of image here". Inside the quotes, you can either use an image URL (which you are probably already familiar with), or a relative path. A relative path means that the image is in the same file directory as your HTML file! To do this, start off with "./", which places you in the same directory as the HTML file, then do the file path! So something like this: src="./Media/LegoBatman.png". So again, here is how the code should be written: <img src="image path here" />

NOTE: When adding images, we generally want to try to add something called "alt text." This is pretty much a brief text description of images and graphics. This helps people who uses screen readers big time, as well as helps with your website's SEO. You should always try to add alt text to your images unless the image doesn't actually contribute to the website's function, such as a background pattern image to make the website look nice. Alt text can simply be added by adding the alt='Alt text here' attribute to the opening tag! Check out this website to learn how to write effective alt text.

| Tables

Compared to the other elements we have in HTML, tables can get a little complicated. Before we get into how to add a table, let's check out how tables are organized. A row is a line of data on a table that goes horizontally. A column is a line of data on a table that goes vertically. The header of a table is at the top of the table, which usually indicates the type of the data being presented. Sometimes, a table might have a footer at the bottom of the table. This is generally used to total the numerical values you see in the table.

To start creating a table, start off with using an opening and closing <table> tag. This just lets the computer know we want to make a table. Next, we want to use semantic tags to help the computer recognize which parts are the headers, which ones make up the table body, and which ones will be the footer. The semantic tags don't have any visual representation, but will help out with stylizing the table when we start using CSS. You can read the Semantic HTML section if you're confused!

If you want to have a table header, inside of the table opening and closing tags, use an opening <thead> element. This is a semantic tag. Inside of the <thead> tags, you'll want to add opening and closing <tr> tags. This element stands for "table row," so it adds a new row! Inside of those tags, (this is the last one I promise) make table header cells by using the <th> element. "th" just means "table header!" Add one for each cell of data you need. For example, if you want three heading cells, "Title," "Description," and "Price," you'll need three <th> elements.

Now, it's time for the body of the table. After the closing <thead> tag, we want to add opening and closing <tbody> tags. Inside of those tags, you can start inputting your data! Inside of the table body element, add a <tr> element for each row that you want. Inside of each <tr> element, you can add <td> elements, which means "table data." So for example, if you want a table with one row, which had three cells of data in that row, you would have one table row element with three table data elements nested inside of it! You can also still use <th> elements in the body if need be.

If you want a footer for your table, instead of using <thead> or <tbody>, you would use <tfoot>! You would input data the same way as the body. To put all of this together, this is what a table would look like in code:

For example, if you want some table data to span two rows, you would do <td rowspan="2">. Finally, you may have noticed in the example above the scope attribute. Some common ones we see are <header>, <body>, and <footer>. I like to view these terms like folders, as they contain stuff from a certain section of that code. Another way you can think about it is like when you are at a grocery store but you need to look for something specific. The store's aisles are labeled to help you look for what you need easily. Here are some benefits you'll find when using semantic HTML:

  • The code will be easy to navigate. Need to find code related to the nav bar? Easy!
  • The website will be easier to stylize with CSS
  • The website's SEO will improve, as well as screen reader usability

List of semantic HTML:

  • <nav>

  • <footer>

| Linking the CSS to the HTML

In order for your CSS to actually work, you need to link it to your HTML file. You can do this through the <link> element. This is a self-closing element, so you don't need a closing tag. Make sure to add this to your <head> element! You'll then want to add an href attribute so you can link to your CSS file. (It works the same as when you are adding an image). That should be it, right? Nope! You also need to add a rel attribute, which stands for "relationship." Basically, it describes the relationship between the HTML file and the CSS file. In this case, since we are linking to a CSS file, we would set the rel attribute to stylesheet. And voila! Your CSS should work now! Here is what the code should look like:

<link href="link to CSS file" rel="stylesheet" />

| Classes and IDs

Classes and IDs pretty much give HTML elements a certain name so you can refer to them in your HTML and CSS. The difference between them lies in their specificity. (I'm still having troubles saying that word...) The specificity order goes in this order, from least specific to most specific:

  1. Element name (p, h1, etc.)
  2. Classes (.classname)
  3. IDs (#idname)
  4. !important

The more specific the selector, the higher priority it gets when it gets assigned a ruleset. As an example, if there were two CSS rulesets that changed the color of h1 elements, but one had an ID name, the one with the ID name would get the priority. Generally, it's good practice to be as least specific as you can - so stick with the element names unless it's more ideal to use a class or ID. This is because if you just go around the CSS giving every class and ID a CSS ruleset, it's gonna be a heckin' pain to style. In fact, things like !important should rarely be used unless you really need to make that element stand out. And when you do use classes and IDs, give them names that are easy to follow.

| Typography

This section will give you the basic rulesets you need to know when you want to deal with typography.

  • font-weight changes how bold or thin the text appears. You can change it to bold or normal. But you can also use "bolder" or "lighter," which changes the font value 1 time higher or lower. It can range from 1 to 1000.
  • font-style can be used to change the font to italics. You can set this to italic or normal.

  • word-spacing is used to set the space between words. While you could use px for this, it's recommended to use em because the sizing will be relative to the size of the font.
  • line-height changes, well, the line height. This can be a unitless number, too!
  • text-align sets how the text is horizonatally aligned within it's container. Some common ones include left, right, center, and justify.

| Colors

There are multiple ways to select a color in CSS. The easiest way is to use named colors. These are just English names for a certain color. However, this can get pretty limited, so if you want access to the widest array of colors available, you can use hexadecimal or RGB values. I'm not gonna dive into how they work here, but hexadecimal values are inputted as #01234A and RGB values are inputted as rgb(01, 02, 03). Which one you use is up to preference! That being said, there is one more way to input a color value, which is through hsl (Hue, Saturation, and Brightness). It is input as hsl(Number value, Percent, Percent).

Also, you can add opacity to the mix! This is referred to as the "alpha" value. You put this in as an extra "a" at the end of rgb or hsl. So, it'll be rgba (0, 0, 0, 0) OR hsla (Number value, Percent, Percent, Number value). Opacity will be a value between 0 and 1, with 0 being completely transparent and 1 being fully opaque. You can do this with hexadecimal as well, but it's a bit awkward, so it's a bit unconventional. You add two numbers to the end of it with 00 being fully transparent and FF being fully opaque.

Oh, I should also point out that there are two ways to change the color of something: through it's color value, or through it's background-color value!

| Box Model

The box model is a framework that indicates how HTML elements are structured and displayed in a browser.



There are four big elements of the Box Model. The content is the actual content of the HTML element. So, maybe it's the text you put in, or maybe it's an image. Next is the padding, which is the space just outside of the content, but inside of the borders of the box. The frame is what it sounds like - it's the frame of the box! And finally, there is the margins. This is the space just outside of the frame. I suppose it would be easier to understand through an example. Take a look at this cool LEGO set based on The Great Wave by Hokusai!



Hopefully this gives you a good idea of how the box model works! If you wanted to change the size of any of these properties, you can do so for any size. While you could use things like margin-left or padding-right, you can more easily set the sizing by using either (ruleset) # # # # (up, right, down, left) or (ruleset) 0 0 (top/bottom, sides)

It's also important to bring up some awkward complications when coming up with specific sizes for a boxed element. Let's say you want a box that's exactly 300 pixels wide and tall. Even if you put in 300 pixels for the height and width, it's gonna be a bit larger because you have to account for the padding and border sizes. Well, it turns out that there are two kinds of box models: the content-box and the border-box! The default is the content box, but if you change it to a border box using the box-sizing ruleset, you can make it so it's easier to get those exact 300 pixels! In this case, you're just resizing the border of the box, so it's easier to get the size you want!

| Display

Display sets whether an HTML element is treated as a block or inline. What are blocks and inlines, you say? Depending on which you use, it may effect the spacing of elements on your webpage. Have you noticed that when you put in something like a heading, the content blocks everything on that line, even if the content doesn't fill up that whole section? That's a block! They Tend to make content go vertical, despite their being enough space on the left or right.

Inline makes it so content can share space in one line, so something like you would see in a navigation bar. Each button element takes up it's own space horizontally, so it's inline! Finally, there is inline block, which takes the best of both worlds!

| Float

Float moves the selected element to the left or right as far as possible within it's container. You use either float: left or float: right! Floated elements must have a width specified. Otherwise, the element will assume the full width of it's containing element, and changing the float value will not yield any visible results.

| Clear

Sometimes, when you use something like float on multiple elements, elements will sorta "bump" into each other, and not allow elements to move in the way you want. Clear sets how elements should behave when this happens. You can set it to left, right, both, or none. This sets it so that whatever side you chose for the element will not touch any other element within the same container.
